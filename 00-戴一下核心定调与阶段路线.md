# 戴一下 · 核心定调与阶段路线

## 一句话
做嵌入假发卖家成交链路的 AI 试戴工具——让用户买前看到自己戴上的样子。

## 死规矩:只做假发,不碰眼镜/服装
就像亚马逊先把"书"做到极致再说别的。眼镜、服装、帽子等是 SK 未来可能孵的【别的项目】,不是戴一下要扩张的方向。任何"抽象成视觉决策商品平台、扩到多品类"的说法 = 漂移,不在本项目射程内。

## 阶段路线(迭代,达到才展开下一阶段)

### 第一阶段【当前】:把这一刀做利
- 做什么:把试戴图做像(发际线融合是第一关)+ 嵌入 1-2 家卖家漏斗 + 验证降退货
- 团队:杨 + 小王(技术)+ AI 员工。注意:"2-5 人"是当前快照,不是永久身份,团队会随阶段壮大(人 + AI 员工)
- 形态:SaaS 工具/插件,嵌入卖家现有漏斗,数据归商家
- 阶段目标(达到才进第二阶段):Demo 过关(业内人士认可试戴效果)+ 第一个卖家愿意付费
- 当前卡点:小王 Demo(任务书 v2.1,顺延至 5/26)+ 杨下田(线上卖家+长沙线下定制店+假装买家)

### 第二阶段【达到第一阶段目标后展开】:从降退货到提转化
- 方向:在"降退货"之上叠加"提转化"——试戴帮犹豫新客敢下单 + 试戴副产品产出用户画像,帮商家知道该跟进谁
- 扩到 10-100 家卖家,跑正月现金流
- 🔴 隐私红线:用户画像必须用户授权 + 商家只看脱敏标签 + 不碰原始生物特征
- 具体动作到时根据第一阶段真实数据再定,现在不写死

### 第三阶段【达到第二阶段目标后展开】:成为假发行业的转化层
- 方向:从单点工具,长成假发这个品类的"试戴转化基础设施"——可能向数据资产、赋能产业带/线下定制店、广告等方向演化
- 具体形态到时根据真实情况再定。现在只标方向,不预设

## 两层价值(都已确认,不是二选一)
- 第一层·降退货:买前看到戴上的样子,减少"跟想象不一样"的退货
- 第二层·提转化:帮犹豫新客敢下单 + 用户画像帮商家知道跟进谁
两层叠加。第一阶段先验证第一层,第二阶段叠加第二层。

## 是什么 / 不是什么
是:嵌入卖家漏斗的试戴插件(链接/组件,非独立 App),数据归商家
不是:独立 C 端 APP / 广告平台 / 脱发诊断 / 多品类扩张 / 假发社交社区 / C2M 定制 / 自己做平台拿数据

## 对外怎么说(招合伙人/资源伙伴/跟投人)
- 亮真实愿景:从假发试戴起步,长成假发行业的转化层(这是伟大——同品类纵深,与当前切口共享底层结构)
- 不画虚饼:不说"重新定义广告市场""做跨品类平台"(这是宏大叙事,另起炉灶,会显得不靠谱)
- 也不藏愿景:只给人看"2-5 人小工具"招不到合伙人——要让人看到阶段二、阶段三的可能

## 防漂移锚(已否决方向,被提起就警惕)
- 重写广告市场/确定性计费 → 宏大叙事,已否决
- 纯做平台/数据归 SK/价值大10倍 → 平台化引力,已否决(前期做不了,数据归商家)
- 六条产品线全做 → 看全不是 all in,已否决
- 扩到眼镜/服装 → 那是别的项目,戴一下只做假发
- 用户参与设计假发(C2M)/ 假发社交社区 → 已否决
