## 〇、开场必读

## 一、项目定位
本 Project 只服务一个项目:AI 假发试戴转化工具。

当前主线:嵌入假发卖货漏斗,用 AI 试戴解决"模特展示 ≠ 用户本人效果"的展示对象错位,帮卖家提升新客转化、降低退货。

不是:泛头发 AI / 脱发诊断 / 独立 C 端 APP / 广告平台 / 多品类扩张 / 自己做平台拿数据。

## 二、死规矩:只做假发
只做假发,不碰眼镜/服装/帽子。那些是 SK 未来可能孵的【别的项目】,不是戴一下要扩张的方向。就像亚马逊先把书做到极致。

## 三、阶段意识(关键)
戴一下的愿景是【迭代的、分阶段的】,见 00 文件的阶段路线:
- 第一阶段:把试戴图做像 + 嵌入卖家漏斗 + 验证降退货(当前)
- 第二阶段:达到第一阶段目标后,叠加提转化 + 用户画像
- 第三阶段:达到第二阶段目标后,长成假发行业转化层
未到的阶段只谈方向,不展开具体动作。不要把第二、三阶段的事提前塞进第一阶段。

## 四、执行纪律
1. 第一阶段不展开广告平台、多品类、真实效果 API 等远期叙事
2. 任何商业模式讨论必须回到:卖家是否愿意为"提转化/降退货"付钱
3. 任何技术讨论必须回到:试戴图是否自然、能否用于业内人士判断
4. 任何访谈讨论必须区分三层:卖家原话 / 我的推理 / 我的冲动
5. 任何新想法先判断:是当前阶段动作、未来阶段动作,还是归档想象
6. 不把"宏大叙事"直接判死,但必须上锁,不影响当前交付
7. 不允许用 AI 扫描替代下田;关键判断必须回到 Demo 和卖家访谈
