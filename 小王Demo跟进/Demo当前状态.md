# 戴一下 Demo 当前状态

更新时间：2026-06-01

## 一、当前目标

当前 Demo 用于验证最小产品闭环：

```text
用户上传正面照
-> 商户上传或拍摄假发商品图
-> 生成用户本人试戴效果图
-> 商户和用户查看结果
```

现阶段优先保证：

- 链路能跑通。
- 页面形态接近真实使用场景。
- 试戴结果尽量保留用户本人身份。
- 商户可以使用自己的假发商品图。

## 二、已经完成

### 1. 静态试戴主链路

已经完成静态发型迁移服务接入，可使用用户照片和参考发型图生成试戴效果。

### 2. 样本测试页面

已经提供内部样本测试台，用于选择用户样本、发型样本、查看结果和记录人工验收意见。

当前本地样本规模：

- 5 位测试用户。
- 11 张用户照片。
- 20 款发型参考图。

测试照片只保存在本地，不上传到公开仓库。

### 3. 手机扫码体验

已经支持扫码进入手机端，用户可上传自拍、选择发型并生成试戴图。

### 4. 商户自助上传页面

已经新增商户自助上传页面：

- 用户上传或拍摄正面照。
- 商户上传或拍摄假发商品图。
- 两张图上传完成后直接生成试戴效果。

### 5. 视频技术路线验证

已经验证 Seedance 2.0 视频任务接口可跑通，但不适合作为当前主产品链路：

- 普通真人照片不能直接输入。
- 真人素材必须先经过火山方舟素材入库和授权。
- 视频生成后人物面部变化较明显。

因此视频能力仅保留为辅助探索方向。

## 三、当前判断

当前最适合继续推进的产品形态是：

```text
面向假发卖家的 AI 试戴工具
```

短期重点不是扩大功能，而是提高静态试戴结果的可验收率：

- 用户脸不能明显变化。
- 发际线需要自然。
- 耳侧和脸颊边缘不能明显穿帮。
- 假发款式需要尽量符合商户商品图。
- 输入失败时要给出清晰提示。

## 四、待办事项

1. 使用更多商户假发商品图验证自助上传链路。
2. 建立固定验收表：身份保持、发际线、边缘融合、商品一致性、耗时。
3. 统计正面照、近景照和边界样本的成功率。
4. 补充隐私说明和图片自动清理策略。
5. 评估抖音小程序挂载商户商品页的最小方案。

## 五、仓库说明

公开仓库不包含：

- API Key。
- 用户真人照片。
- 未确认授权的真人素材。
- 生成结果图。
- 本地运行日志。

这些内容只保存在本地测试环境或单独的私有存储中。
