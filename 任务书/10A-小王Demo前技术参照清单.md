# 10A · 小王 Demo 前技术参照清单

## 这份清单怎么用

- **它是扩眼界,不是调研任务。** 做 Demo 前花半天扫一眼,心里有个底——别人做到什么程度、技术走哪条路。
- **看完只交付一句话**(不写报告):
  > 「假发试戴建议走哪条技术路线?(纯商业 API / 商业 API + 开源模型微调 / 3D 方案)为什么?预计能做到什么效果?」
- **看是为了判断「假发该怎么做」,不是「抄哪个做得好」。** 看太多同类容易不自觉模仿,反而做不出 SK 自己的判断。

---

## 一、同类产品(看效果标尺 + 交互,不写竞品报告)

看每个产品时,重点只看三件事:① 发际线融合是否自然;② 是否支持商品页 / API / SDK 嵌入;③ 它走 3D / AR 路线,还是生成式图像路线。

**1. 弥知科技 Kivisense**(tryon.kivisense.com)
- 国内做 AR 试戴插件最成熟的,订阅制、能接 Shopify,在 Shopify 应用商店有上架
- 重点看:它是 **3D 刚体方案**(眼镜 / 鞋 / 珠宝),不是图像生成。判断——假发能不能也走 3D?还是必须图像生成?这是路线分叉
- 顺便看它怎么打包成插件(SK 未来形态的参照)
- ⚠️ 待验证:它当前是否仍主推试戴(有迹象转向 AI 营销素材生成)

**2. ModiFace**(modiface.com,欧莱雅旗下)
- 头发虚拟试戴可试不同发色,能嵌商品页、做前后对比,有 SDK 和小程序
- 重点看:它是行业标杆,**但实时功能有时发际线融合不自然**——连欧莱雅级别都没解决发际线融合,这正是假发最难的点。**看它的失败处比成功处更有用**

**3. Perfect Corp / YouCam**(yce.perfectcorp.com)
- 用生成式 AI 做发型试戴,单次 API 调用可视化发型,**有现成 API**
- 重点看:它走**生成式 AI 路线**(和小王可能同路),且提供 API。可直接试它的 API——**如果效果已够好,SK 可能直接调它,不用自己做**

**4. Ulta Beauty 的 GLAMlab**(美国零售商)
- 用英伟达 AI,自拍 / 上传图试发型变换,结果较真实,**明确说要扩展到假发(wigs)品类**
- 重点看:这是**唯一明确提到要做「假发」的大玩家**。留意它——如果 Ulta 真做假发试戴,是未来的直接对标 / 竞争

## 二、头发 AI 模型(只判断能否服务假发 SKU 试戴)

这些是任务书附录 C 已记的开源模型,作为**技术雷达**了解(不是要求跑通全部):

- **HairFastGAN**(发型迁移,P0 优先)
- **Stable-Hair**(扩散模型,先去原发再贴新发)
- **ComfyUI 的 StableHair 节点**(可视化快速试)

核心判断:这些是「换发型」模型(改你自己的头发),**假发试戴是「贴一个具体商品 SKU 上头」——两件事不完全一样**。换发型模型能不能改造成「把指定假发 SKU 戴到用户头上」?

口径(与 v1.8 一致):开源模型作为**技术雷达**,与商业 API 平行了解。若商业 API 横评无法产出可用于业内判断的样图,小王再评估 HairFastGAN / Stable-Hair 是否值得另开 3–7 天探索。**当前不要求跑通全部开源项目,不进入训练 / 微调 / 自研。**

## 三、商业 API(主线,先试能不能出图)

按 v1.8,商业 API 是主线。重点不是 API 名气,而是:**同一张人像 × 同一款假发,谁出的图最自然。**

- 国内:通义万相、即梦、可灵
- 国外:Perfect Corp 的发型 API(见上)、fal、Replicate

## 四、看的时候守三条

1. **别信这张清单是对的。** AI 推荐的 APP 常有过时或编造的。每个都自己验证:查应用商店更新日期、看最近一星差评、搜「APP 名 + 不能用」。确认它还活着、真能用,再花时间研究。
2. **重点盯「发际线融合」。** 上面所有产品最容易露馅的都是同一处:假发 / 新发和真实发际线的衔接。连 ModiFace 都没完全解决。做 Demo + 盲评时,**先看发际线**——这是判「像不像」第一关。
3. **看完只回答一个问题。** 不写报告。只答:假发试戴建议走哪条技术路线?为什么?预计能做到什么效果?
