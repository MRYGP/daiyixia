# 10B · 小王 AI 原生工作方式与竞品学习清单

## 一、本文件是什么
不是新增产品需求,是回答一个问题:小王如何用 AI 把自己临时放大成一个技术 Demo 小组?
核心:**AI 给小王补位,不是让小王去管理一个 AI 团队。** 你的任务始终是把试戴图做像,AI 只是帮你做得快、记得清的工具。当前阶段(见 00 文件)只验证:试戴图像不像、商品图保不保真、正侧面能否稳定生成。

## 二、6 个 AI 员工(阶段一启用)

### 1. AI 架构秘书 ⭐(最该先建)
建一个 AI_CONTEXT.md,每次让 AI 写代码前先读它,防止 AI 每次重新推导、代码漂移。只写 4 件事:
- 当前目标:5/26 前做可演示假发试戴 Demo
- 不做:登录/支付/后台/小程序/用户画像/广告/商品库/批量上传/多品类
- 核心流程:上传假发商品图 → 上传正面/侧面人脸 → 生成试戴图
- 验收:发际线自然 / 光照融合 / 商品图保真 / 正侧面一致 / 失败可解释
(技术栈、数据隐私、每日变更等需要时再加,先别堆满)

### 2. AI 副工程师
派单时只给单任务,加一句约束:"你只能完成本任务,不允许新增产品功能;任何超出范围的建议写到'后续建议',不要写进代码。" 防止 AI 顺手把登录/后台/SKU 加进来。

### 3. AI 图像实验员 ⭐(这是核心活,不是额外活)
结构化记录每次图像实验——这直接喂给雷达图,证明"能稳定控制"而非"偶然跑出一张好图"。每次记:技术栈 / 人脸图(正侧/光线) / 商品图类型 / prompt / 参数 / 耗时 / 成功否 / 自然度1-5 / 商品保真1-5 / 人脸保持1-5 / 正侧一致1-5 / 失败原因(发际线/光照/角度/商品跑偏/速度) / 结论(留/淘/复测)。每个主栈至少留 5 条,最终选栈要能追溯到记录。

### 4. AI QA 盲评秘书
把结果图匿名编号(A1/B1…),不告诉评审哪个模型、不暗示偏好,只让评审打分。盲评 6 问:整体自然 / 发际线假不假 / 像同款商品假发吗 / 人脸变形吗 / 正侧像同一人同一款吗 / 你若是卖家愿不愿看第二版(是否)。优先评审:假发卖家/电商/MCN/买过假发的人;朋友只做补充。AI 只整理,审美最终判断归真人(尤其业内)。

### 5. AI 安全隐私检查员(Demo 给真人试用前才需要,即 5/26 后;现在知道风险在哪即可)
6 问检查表:上传图是否保存 / 是否传第三方 API / 第三方是否留存训练 / 总结里是否含未授权真人原图 / API Key 是否暴露前端 / 是否有删除测试图的方法。未知就写"未知",不假装清楚。当前不做重合规。

### 6. AI 日报秘书
复用 10 号任务书已规定的每日 21:00 200 字日报格式(今日完成/卡点/明日/需杨协助/风险)。卡住超 2 小时直接微信杨,不等日报。

## 三、建议的每日节奏(参考,非强制)
小王自行安排,大致:早上定 1 个 P0 → 上午让副工程师写一个模块 → 下午图像实验员跑图记录 → 傍晚盲评秘书整理样图 → 晚上日报。**这是建议节奏不是打卡制,小王按自己习惯调整。**

## 四、竞品只学什么(不做竞品调研,竞品调研归杨)
小王只看"对当前 Demo 有用的入口形态 + 技术难点",不写竞品报告。
- 具体同类产品清单 + 看的纪律 → 见 `10A-小王Demo前技术参照清单.md`,不在此重复
- 一条关键技术难点(补充 10A):侧脸/耳朵/脖子/发际线是穿帮重灾区,正面好不够,侧面一崩就成娱乐滤镜;**发色试戴 ≠ 假发试戴**(发色保留原结构,假发要重建发型体积和边界,难一个量级,别用发色做得好推断假发也能做好)
原则:学"入口轻、结果快、绑定具体商品";不学会员/商城/后台/营销/analytics/多品类。

## 五、阶段性启用规则
- 阶段 1(5/26 前):只启用上面 6 个。目标=Demo 能演示、雷达图能交付、失败也成判断资产
- 阶段 2(Demo 过关+业内认可后):再加 AI 反馈分析员 / 插件工程师 / 数据埋点员 / 相似度验证员
- 阶段 3(第一家卖家愿试用后):再加 AI 客服收集 / 版本经理 / 成本会计 / 接入文档员 / ROI 记录员
**现在不许提前启用阶段 2/3 的员工。**

## 六、防漂移规则
出现以下立刻暂停:从假发跳眼镜/服装 / 从 Demo 跳完整 SaaS / 从试戴跳广告平台 / 从技术判断跳全公司组织设计 / 从商品图上传跳 SKU 后台 / 从试戴跳用户画像 / 从竞品学习跳竞品报告。
判断句:**这件事是否直接帮 5/26 Demo 过关?** 不能 → 放"后续候选",不进本周。

## 七、给小王一句话
这份文件不是让你多干活,是让你少被 AI 带偏。5/26 前你只用 AI 帮你:写代码、跑图、记录实验、做盲评表、查隐私、写日报。你不做:商业判断、产品经理、用户访谈、竞品报告、后台、小程序、广告、用户画像。
最重要的交付不是"Demo 多漂亮",而是回答:**假发试戴这件事,小王用 AI 加持后能做到哪一步?哪步需要补人/补资源/补时间?**

## 八、与 10 号任务书的关系
10 号定:目标/边界/交付物/验收/失败兜底。本文件只补"怎么用 AI 工作",不改 10 号任何硬交付。
