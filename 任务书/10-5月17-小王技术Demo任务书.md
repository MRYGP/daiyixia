# 10 · 小王技术 Demo 任务书

## 第一层·主任务书(先看这个)

### 5 天目标(第一版·过关线版)

让商家能打开链接，上传假发商品图 + 正面照 + 侧面照(各一张)，看到自己从正面和侧面戴上该款假发的效果。  
**目标**:超出镜子里能看到的——用户在镜子里只能看到正面，我们要让他们看到正面+侧面+(V2 探索后脑勺)。  
**V1 验证**:正面+侧面能否都过**自然度**门槛。  
**只验证「AI 能不能做」**,不验证卖家愿不愿付费,也不追求体验完美。

### 第一版 vs 第二版:两版分工

| 维度 | **第一版·过关线版**(5/22 硬交付) | **第二版·直觉版**(5/22 后,本周不做) |
|---|---|---|
| 目的 | 让杨拿去见业内人士,做技术可行性判断 | 业内人士确认方向后,做能给真人用的产品 |
| 接入 | 网页或扣子 Bot,能打开就行 | 移动小程序 / 手机网页,扫码即用 |
| 拍照 | 上传照片 | 调起摄像头实时拍照 + 拍照引导 |
| 假发选择 | 商家上传自己的假发商品图；V1 选 2-3 款做多角度测试(正面+侧面)，优先选颜色/长度/刘海特征明显的款，便于验收商品图保真度 | 6+ 张缩略图分类(自然/时尚/短/长/烫染/老人款) |
| 加载提示 | 「加载中」 | 「AI 正在为你戴上...(约 15 秒)」 |
| 出图后 | 显示一张图 + 再试一次 | 显示图 + 再换一款 + 保存图片 |
| 速度 | 30 秒内 | 15-20 秒(扫码到出图 30-45 秒) |
| 自然度 | 盲评 ≥ 3.5 | 盲评 ≥ 4.0 |

**第一版做不到第二版的事**,**不是失败**。第二版需要等业内反馈,不能凭空设计。

**唯一例外**:第 1 天选主栈时,如果两个栈质量接近,**优先选能为第二版铺路的**——即支持移动端调用、响应时间较快(<20 秒)、有 API 可程序化调用——但**不为此牺牲第一版的过关质量**。

2. 给看 3 张图(主栈,样本 A/B/C 各 1 张最佳输出)
3. 回答主任务书过关线的 **3 个问题**
4. 记录答案,不引导

### A5·海外 API(举例:Replicate / fal.ai)

- 国内可能需要代理;**没代理可跳过**,记「未测试」,不必为此专门折腾
- 有代理:各跑 1–2 次即可

### A6·扣子(Coze)起点(举例)

- 地址:[https://www.coze.cn](https://www.coze.cn)
- 先在**插件市场**搜「AI 换发型」「图像编辑」;有现成插件直接用
- 没有则:**图像生成节点 + Prompt**,示例:

  > 将这张照片中人物的发型替换为[假发款式描述],保持面部和五官不变

- 输出不行 = **扣子不适合假发试戴**(有效结论),换下一栈

### A7·「成功」的技术定义(举例,与自然度分开)

**成功** = 以下 4 条全满足(像不像另算):

1. API/页面调用**没报错**
2. **30 秒内**返回(超时 = 失败)
3. 输出图**能正常打开**
4. 图里能看到**人头 + 某种发型**(不管自然度)

**「5 次至少 4 次成功」**指上述 4 条;自然度用盲评 1–5 分单独评。

### A8·5/22 交付方式(举例)

**微信发杨**(中午 12:00 前):

1. **一个在线文档**(Notion / 飞书 / Google Doc),含:

   - 盲评结果(3 人 × 主栈;备选栈可简写)
   - 500 字技术总结(含 API 是否留存照片,查不到写「未知」)
2. **Demo 链接**(活链)
3. **2 分钟录屏**(微信文件,一个完整视频)

失败兜底:文档里写失败 5 项 + 3 张图 + 说明录屏,无 Demo 链接可接受。

## 第二层·附录 B·边界与原则

### B1·禁止行为(Demo 阶段不做)

❌ 美化 UI(配色/字体/动画)  
❌ 用户登录/注册  
❌ 管理后台  
❌ 付费/支付  
❌ 加载动画优化  
❌ 自定义主题  
❌ 分享/社交  
❌ 历史记录  
❌ AI 推荐  
❌ 商品库 / SKU 管理 / 商家后台 / 批量上传 / 历史记录(V1 只加一个商品图上传 input)

### B2·允许行为

✅ 极简表单:选假发 + 上传照片 + 点生成  
✅ 简单「加载中」  
✅ 显示一张结果图  
✅ 「再试一次」按钮  
✅ 报错时简单提示(不必很友好)  

### B3·「卡住 2 小时」具体动作

| 场景 | 动作 |
|---|---|
| API 注册失败 / 文档看不懂 | **跳过该栈** |
| 生成完全不对题 | 改 1 次 Prompt,仍不行 → **跳过** |
| 扣子工作流跑不通 | **换下一栈**,不深挖 |
| 5 栈都不行 | **立刻走失败兜底**,不继续硬试 |

任何场景 **>2 小时无进展** → 微信杨(问题 + 截图),不需要先找到答案。

### B4·失败兜底交付物(5 栈全不过关)

1. 最接近可用的 **3 张**图  
2. **失败原因排序**:发际线 / 光照 / 角度 / 稳定性 / 成本 / 接入难度  
3. 是否存在「只适合正面照」的窄场景  
4. 最可能继续的 **1 个**技术方向 + 预估时间  
5. **下一步建议**:继续试错 / 半人工 Demo / 暂停方向  

### B5·隐私边界

- 测试照须**本人同意**;Demo **不保存**上传照(或阅后即焚,总结里写明)  
- 对外展示只用授权图或 AI 脸  
- 总结里不放未授权真人原图  
- 第三方 API 记录照片是否被平台留存/训练;查不到标「未知」  

### B6·自然度过关线(第 2 天主栈)

- 3 人盲评平均自然度 **≥ 3.5**  
- **≥ 2/3** 愿意继续试第二款  
- 技术「成功」:**5 次 ≥ 4 次**(见附录 A7)  

**商品图保真度(V1 新增)**:生成结果须保留上传假发商品图的核心特征——颜色、长度、刘海/分缝、卷曲度、蓬松度。自然但不像原商品,不算完全过关。

### B7·P0 五个技术挑战(第 2 轮前 2 栈标注)

| # | 挑战 | 失败长什么样 |
|---|---|---|
| 1 | 发际线贴合 | 硬边、悬浮、发际错位 |
| 2 | 头型与角度(B/C) | 侧脸穿帮、耳后空白 |

| 6 | 多角度一致性(同一款假发正面+侧面结果风格统一) | 两张图明显是「不同假发」 |
