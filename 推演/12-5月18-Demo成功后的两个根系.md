# 头发产品推演(12)·Demo 成功后的两个根系

## 一、起点:杨 5/18 早的判断

杨提出:"不管是假发、植发、接发、理发,广告的场景都存在。如果新的广告是试戴呢?"

并澄清判断的真实结构:"我们第一代做出来 AI 试戴,效果好的话,作为厂家、店铺,开拓新客户、维护老客户是一个功能。另一个就是直接把原来的广告改为让用户看到自己用后的效果。这二方面是这个产品可以生出来的根系。"

**关键澄清**:根系想象 ≠ 换方向。第一代 Demo 仍然是"用户拍照试戴假发",目标仍然是 5/22 见业内人士。但 Demo 跑通后,这件事天然能长出两个分支。

## 二、根系结构

```
        第一代 Demo
    (用户拍照试戴假发)
            ↓
若 Demo 效果好 + 业内人士认可 →
↓
这件事可以长出两个根系:
│
┌───────┴───────┐
↓               ↓
根系 A           根系 B
卖家工具       试戴即广告
│               │
厂家/店铺      - 把假发/植发/理发
开拓新客户      的传统广告位 →
维护老客户       试戴广告位
SaaS/分佣      - 用户在广告里
看自己用后效果
- CPT/转化分成
- 付费方扩展到
任何想做试戴
的商家
```

两个根系**不冲突**:
- A 服务"已经卖假发的商家"——窄而深
- B 服务"任何想用试戴做广告的商家"(假发/植发/接发/理发/美瞳/眼镜)——宽而浅

**演化路径**:Demo 跑通 → 先做 A 站稳现金流 → 再扩 B 拉规模

## 三、Kimi 的推演框架(标注来源)

5/18 早杨把这个洞察发给 Kimi 推演,Kimi 输出"试戴即广告"完整商业模型。

### 1. 传统广告 vs 试戴广告的逻辑差异

**传统广告**:品牌付费 → 用户看到"别人的样子" → 用户想象"自己的样子" → 搜索/对比 → 到店 → 购买

**试戴广告**:品牌付费 → 用户上传照片 → 用户看到"自己的样子" → 决策在广告内完成 → 直接购买/预约

**Kimi 命名**:广告从"信息传递媒介"升级为"决策完成工具"。

### 2. 与 SK 现有理论的连接

| 模式 | 原案例 | 在试戴广告中的映射 |
|---|---|---|
| P1 产出物即分发 | Gamma | 用户试戴生成的图本身就是广告内容,分享即传播 |
| P6 意图续航 | Spangle | 试戴广告不需要落地页,意图在广告内直接完成闭环 |

### 3. 四方价值网络(Kimi 估算,未验证)

- **用户侧**:决策信心从 30 → 85
- **商家侧**:转化效率从 5% → 28%
- **平台侧**:停留时长从 15 → 55
- **SK 侧**:每次试戴 = 一次结构化数据输入

### 4. 三种商业路径

| 路径 | 性质 | 收费模式 |
|---|---|---|
| A·嵌入现有广告系统(轻) | 试戴 SDK 嵌入抖音/小红书/淘宝 | CPT 或转化分成 |
| B·独立试戴广告平台(重) | 自建"试戴广告市场" | 广告位竞价 + 交易佣金 |
| C·B 端 SaaS(中) | 给假发/植发/理发机构 | 月费订阅 |

**Kimi 判断**:路径 A 最轻,路径 C 最可控,路径 B 太重不适合当前阶段。

### 5. 错位母模型应用(Kimi 借 DOC-D030 重新定义需求)

| 场景 | 用户说的 | 用户真正雇佣的 |
|---|---|---|
| 假发 | "我想买假发" | "我想看起来像自己,但更好" |
| 植发 | "我想植发" | "我想知道做完还像不像我" |
| 接发 | "我想接发" | "我想长发但不损伤自己头发" |
| 理发 | "我想剪头发" | "我想剪完不后悔" |

**Kimi 元命名**:用户不是在雇佣"头发产品",是在雇佣"一个不会后悔的自己"。试戴广告卖的不是产品信息,是"后悔保险"。

## 六、风险与未验证假设

**Kimi 推演里的未验证假设**:
