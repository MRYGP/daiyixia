## 一、KreadoAI 真实定位重新确认(重要修正)

**KreadoAI 不是 SK 的竞争对手,是相邻领域**:
- KreadoAI 主线 = **AI 营销素材生产平台**(数字人视频 + AI 模特商拍图 + 多语言广告素材)
- KreadoAI 客户 = 跨境电商卖家**做营销内容**(不是给消费者用)
- SK 头发产品要解决的 = **"用户怎么决定买不买"**(消费者决策辅助 + 卖家转化工具)
- 两者**完全不在同一赛道**

**5/17 把 KreadoAI 当成"赛道占位者"是错判**。
它甚至可能是 SK 的潜在合作对象(它有卖家客户,SK 有试戴能力)。

## 二、技术上"植入卖家页面"的 3 种方式

### 方式 1·JS SDK(JavaScript 嵌入)
- 卖家把 1 段 JS 代码放到自己网站
- 用户点击 → SDK 调用 SK API → 显示试戴结果
- **优点**:卖家 5 分钟接入,跨平台(Shopify/独立站/H5/小程序内嵌网页)
- **缺点**:微信/抖音/淘宝小程序原生不能直接跑 JS

### 方式 2·小程序原生组件
- 微信/抖音/淘宝各写一套
- **优点**:平台内体验流畅
- **缺点**:3 倍工作量,SK 资源不够

### 方式 3·后端 API
- SK 提供 RESTful API,卖家自己写前端
- **优点**:SK 工作量最小,适合大客户
- **缺点**:小卖家做不到,SK 拿不到用户交互数据

## 四、推敲结论

**SK 头发产品最可能先做成 = JS SDK + 后端 API 双轨**

**理由**:
- 2-5 人团队能支撑
- 接入快(JS SDK 5 分钟,API 1 天)
- 跨平台覆盖
- 数据飞轮起得来
- 大厂不做这种轻量嵌入
- 商业模式清晰(订阅 + 调用付费)

## 五、明确不做的形态(防漂移)

| 形态 | 不做原因 |
|---|---|
| 独立 C 端 APP | 流量贵,留存差,小公司养不起 |
| 独立 SK 品牌微信小程序 | 同上 + 推广受限 |
| 多品类视觉决策平台(假发→眼镜→帽子) | VC 故事写法,路径 B 不走 |
| 同时做 3 个小程序原生(微信+抖音+淘宝) | SK 资源不够,先 JS SDK 跑通再说 |
