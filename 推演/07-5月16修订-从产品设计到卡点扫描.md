# 头发产品推演(07)· 5/16 修订:从产品设计到卡点扫描

## 一、5/15-5/16 推演的根本性错误

5/15 一整天的推演(从假发 ToC → 假发 ToB → 私密化身 → 三入口模型 → 第一代 MVP 化身管理)走的是**"设计独立产品拉新用户"**路径。

杨在 5/16 下午明确指出错误:

> "你做的事情是开发假发产品。而我们要做的是什么?解决假发准用户为什么没买假发,或者说让假发准用户成为真正的用户的转化率就够了"

> "SK 库的独特理念(错位/两层断层/多维思考)是用来洞察卡住的人和场景的,不是用来设计产品的"

> "我们要在现有的用户流、现有的模式上去嵌入、寄生,而不是去开拓用户。我们的产品后面肯定是要有用户拓展,但我们这里是说,用户的准用户"

**两件事的根本区别**:

| 维度 | 5/15 路径(错) | 5/16 路径(对) |
|---|---|---|
| 起点 | 假发市场 = 待切入的空间 | 假发市场 = 一群正在卡住的准用户 |
| 用户定义 | 待寻找(猜测脱发/化疗/时尚) | 已存在(正在搜假发但没下单) |
| 产品定义 | 待设计(独立 APP) | 待发现(他们卡在哪一步,补哪一步) |
| 工具用途 | 筛选产品方向 | 扫描漏斗卡点 |
| 商业模式 | 拉用户分佣 | 漏斗所有者付费 |
| 用户来源 | 我要拉新 | 已经在淘宝/抖音/小红书 |

## 三、用铁律 19 重新看头发市场

按"卡点扫描"姿态走一遍:

### 现有漏斗(中国假发买方漏斗)

```
有念头(头发不满意)
↓
搜索(淘宝/抖音/小红书)
↓
进店铺/直播间
↓
看商品(看模特图 / 看评价 / 看详情)
↓
犹豫(加购、收藏、不下单)  ← 准用户在这里
↓
离开 / 回头
↓
下单
↓
收货 → 试戴
↓
保留 / 退货(假发行业退货率 25-40%)
```

### 用 SK 工具扫描每一层的卡点

| 卡点 | 在漏斗哪一层 | 痛感强度 | AI 能解 | 卖家付费意愿 |
|---|---|---|---|---|
| **展示对象错位**(模特 vs 自己) | 看商品 → 犹豫 | 极高 | 能 | 极高 |
| **戴不好的退货** | 收货 → 试戴 → 退货 | 极高 | 部分能 | 极高 |
| 售后焦虑 | 犹豫 | 中 | 部分能 | 中 |
| 选择困难(SKU 爆炸) | 搜索 → 进店 | 中 | 能 | 低 |
| 复购记忆 | 复购 | 低 | 能 | 低 |

### 第一刀切哪个卡点

**卡点 1:展示对象错位(模特 vs 自己)**

- 用户的不可见预期:"模特戴的我戴会不会只是卖家秀"
- AI 杠杆:用户上传照片 → 看自己戴上的效果
- 卖家付费理由:转化率提升 = 直接多卖货
- 这是 5/15 推演中已经被识别但当时定位错了的卡点

## 四、第一代 MVP(5/16 修订版)

**核心动作**(只做 1 件事):
- 用户即时拍一张自己的照片
- 看到自己戴各种假发的样子
- 没了

**砍掉所有 5/15 的复杂功能**:
- ❌ 化身管理(过度抽象)
- ❌ 数据脱敏(后续优化,第一代不做)
- ❌ 多轮对话调整(第二代加)
- ❌ 三入口模型(过度规划)
- ❌ 跳转购买(第一代不接入商家,先证明用户愿意用)

**第一代的唯一验证目标**:用户愿不愿意上传自己的照片来试戴假发?

- 愿意 → 第二代加多轮对话、加商家跳转
- 不愿意 → 推翻重做

**第一代的寄生位置**(铁律 19 要求):

不是嵌入卖家店铺(那是第二代)。第一代寄生在:
- **小红书/抖音的内容自传播**——用户拍出"我戴假发"自然分享 → 朋友看到来试
- **微信群分享**——脱发群/化疗群里互传

第一代不解决"如何盈利",只解决"用户愿不愿意用 + 愿不愿意分享"。

## 五·5/17 新方向涌现:从「新客转化卡点」到「老客粘性卡点」

> **定位**:不替换 5/16 方案,作为**并行验证方向**。5/16 主线(新客转化 / 展示对象错位)继续走;本节约 5/17 杨的业务直觉切换,用 5/17 卖家访谈 **Q6-Q9** 验证是否成立。

| 维度 | 5/16 主线(仍有效) | 5/17 并行方向(待验证) |
|---|---|---|
| **卡点位置** | 决策犹豫层(看图 → 加购不下单) | **留存复购层**(买过的人为何不再买 / 为何流失) |
| **产品形态** | 卖家店铺工具(淘宝/抖音店铺内试戴) | **卖家自有 APP/网站的 SDK / API / 插件** |
| **数据飞轮** | 弱(每次新客从零上传) | **从弱变强**(老客头型 / 偏好 / 历史试戴沉淀在卖家侧) |
| **SKILL 对齐** | 部分对齐(视觉决策 + 转化) | **与 `core/SKILL-视觉决策商品的AI试戴层.md` 完全对齐**(复购场景 + 1.5 层场景→SKU 匹配) |

**为什么 5/17 认为「嵌入卖家自有 APP」更接近铁律 19**:
- 淘宝/拼多多店铺 = 平台漏斗,卖家只有流量,**没有用户资产**
- 卖家自有 APP / 小程序 / 会员站 = 卖家已沉淀老客,头型与购买史已在卖家手里 → **数据飞轮天然存在**,每多试一次试戴质量越好
- 付费逻辑也可能从「帮新客转化」转向「帮老客复购 / 降流失」——卖家为 LTV 付费,而非单次拉新

**与 KreadoAI(08 文件)的关系**:KreadoAI 做广告素材,不做卖家自有渠道嵌入;本方向若成立,竞品空白可能更大,但天花板警示(08 第五节)仍适用。

**验证方式(5/17 卖家访谈 Q6-Q9)**:
- Q6:老客户占比多少?复购率大概多少?你最想提升的是新客转化还是老客复购?
- Q7:你有没有自己的 APP / 小程序 / 会员网站?老客主要在哪跟你互动?
- Q8:如果试戴能力能以插件/SDK 嵌进你的自有渠道,你愿意接吗?谁来做技术对接?
- Q9:老客第二次买假发,卡在哪?(款式腻了 / 忘了自己适合什么 / 懒得再挑)——AI 试戴+历史偏好能解吗?

**决策规则**:
- Q6-Q9 强信号(≥3 家卖家明确愿为复购/自有渠道付费) → 5/18 起与 5/16 主线并行做第二轨 demo 规划
- Q6-Q9 弱信号 → 本节约档,全力 5/16 新客转化主线

## 七、5/16-5/17 修订后的行动计划

### 5/16-5/17:卖家访谈 + 漏斗卡点验证

**主战场:8-12 家假发卖家**(不再是消费者)
- 淘宝 / 拼多多 / 1688 / 抖音直播间
- 不同规模:月销 100 单 / 月销 5000 单 / 月销 5 万单
- 不同品类:医疗用 / 时尚 / 老年 / cosplay

**卖家访谈核心 5 问**:
1. 你店铺的转化率是多少?(进店访客 → 下单)
2. 用户最常在哪一步流失?(看图 / 看评价 / 加购 / 结算)
3. 用户加购但没下单,最主要原因是什么?
4. 如果有一个 AI 工具让用户在你店铺里直接试戴假发,你觉得能提升转化多少?
5. 转化提升 30-50% 你愿意付月费多少 / 分成多少?

**5/17 追加·老客粘性方向(Q6-Q9,验证第五节并行方向)**:
6. 老客户占比多少?复购率大概多少?你最想提升的是新客转化还是老客复购?
7. 你有没有自己的 APP / 小程序 / 会员网站?老客主要在哪跟你互动?
8. 如果试戴能力能以插件/SDK 嵌进你的自有渠道,你愿意接吗?谁来做技术对接?
9. 老客第二次买假发,卡在哪?AI 试戴+历史偏好能解吗?

**副战场:从卖家手里要 3-5 个真实买家访谈**
- 加了购物车又取消的:为什么犹豫?什么让你最终没下单?
- 已购买的:你犹豫过吗?最后什么让你下单?
- 退货的:为什么退?
