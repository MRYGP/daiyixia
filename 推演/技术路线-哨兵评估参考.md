# 哨兵对"技术路线兜底文档"的评估

## 一、哨兵的核心判断

那份原文档表面克制("先 API,不行再开源,不做训练"),
但实际上**扩大了小王任务书的工作面**——
让小王在 1 个主栈外还要研究 3-5 个开源项目作为兜底,
"2 小时实验 HairFastGAN"实际需要 6-12 小时。

## 二、哨兵能确认的事实

- HairFastGAN(AIRI 出品)做发型迁移 ✅ 真实存在
- Stable-Hair 扩散模型先去原发再贴新发 ✅ 流程描述准确
- HairCLIP / Barbershop 偏研究 ✅ 训练数据印证
- 商业 API 适合快速验证 ✅ 通用工程经验

## 三、哨兵不能确认的事实

- HairFastGAN "1 秒级推理"——可能但需实测
- 通义万相 / 即梦 2026 年 5 月真实质量——无数据
- ComfyUI_StableHair_ll 节点真实可用性——无数据

## 四、哨兵给小王任务书的 50 字附录建议

> 如果商业 API 横评 5 个全部不过关,先告诉杨,
> 杨决定是否启动开源模型兜底路径(HairFastGAN 优先,
> 但需另算 3-5 天工程化时间)。**小王不主动研究开源模型**。

## 五、关键时间节点

- **5/22 之前**:商业 API 一条道走到黑
- **5/22 之后**:如果 API 自然度不够 + 业内人士说"质量不够",启动 HairFastGAN 探索(1-2 周 + 可能需要外包)
